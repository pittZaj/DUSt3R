## 🐛 问题1: 体积计算报错

### 错误信息
```
❌ 体积计算失败: unsupported format string passed to NoneType.format
```

### 根本原因

在优化尺度标定方法时，我将标定基准从"投影边界长度+宽度"改为"最大跨度"，但：
1. 删除了`self.ground_real_width`的计算
2. 体积计算报告中仍然引用`{self.ground_real_width:.2f}`
3. 导致格式化字符串时出现`NoneType.format`错误

### 修复方案

**修改1**: 设置`self.ground_real_width = None`
```python
# coal_pile_ply_analyzer.py, line 353-356
self.ground_real_length = real_length
# 不再自动计算宽度，因为最大跨度是唯一的
# 但为了兼容体积计算报告，设置为None
self.ground_real_width = None
```

**修改2**: 更新体积计算报告
```python
# 旧版本（会报错）
✅ **已使用尺度标定**：地面真实长度 {self.ground_real_length:.2f}m（输入），宽度 {self.ground_real_width:.2f}m（自动计算）

# 新版本（修复后）
✅ **已使用尺度标定**：最大跨度真实长度 {self.ground_real_length:.2f}m（用户输入）
```

**修改3**: 更新尺度标定结果表格
```python
# 旧版本
| 地面尺寸 | ... | {self.ground_real_length:.3f}m × {self.ground_real_width:.3f}m |

# 新版本
| 地面尺寸 | ... | {self.ground_real_length:.3f}m（最大跨度） |
```

### 代码验证

在`calculate_boundary`方法中，确实正确使用了地面平面：

```python
# coal_pile_volume_processor.py, line 3826-3843
if self.ground_plane is not None:
    a, b, c, d = self.ground_plane  # ✅ 使用地面识别的平面

    # 计算所有点到地面平面的垂直距离
    distances = (a * points[:, 0] + b * points[:, 1] + c * points[:, 2] + d) / np.sqrt(a**2 + b**2 + c**2)

    # 堆顶位置：距离地面最远的点
    max_dist_idx = np.argmax(np.abs(distances))
    peak_position = points[max_dist_idx].tolist()
    pile_height = float(np.abs(distances[max_dist_idx]))

    # 地面高度：中心点在地面平面上的Z坐标
    center_xy = points[:, :2].mean(axis=0)
    ground_z = float(-(a * center_xy[0] + b * center_xy[1] + d) / c)

    # 将所有点投影到地面平面上（垂直投影）
    normal = np.array([a, b, c]) / np.sqrt(a**2 + b**2 + c**2)
    projected_points = points - distances[:, np.newaxis] * normal
```

**结论**: ✅ **完全使用地面识别的地面平面，没有重新计算地面**
