# horizontal_section 方法详解

## 📐 核心原理

### 数学基础：水平截面法（Horizontal Section Method）

水平截面法基于**积分原理**，将不规则三维体积分解为多个水平薄层，然后累加：

```
V = ∫[0→H] A(h) dh
```

其中：
- `V` = 总体积
- `H` = 煤堆最大高度
- `A(h)` = 高度h处的横截面积
- `dh` = 微小高度增量

### 离散化实现

由于点云是离散的，我们使用**梯形法则**（Trapezoidal Rule）进行数值积分：

```
V ≈ Σ[(A[i] + A[i+1]) / 2 × Δh]
```

其中：
- `A[i]` = 第i层的截面积
- `Δh` = 层厚度
- 梯形法则比矩形法则更精确

## 🔧 实现细节

### 步骤1: 计算垂直距离（使用地面平面）✅

```python
# 统一使用地面识别中找到的地面平面
if self.ground_plane is not None:
    a, b, c, d = self.ground_plane  # 平面方程: ax + by + cz + d = 0

    # 计算每个点到地面平面的垂直距离（带符号）
    distances = (a * points[:, 0] + b * points[:, 1] + c * points[:, 2] + d) / np.sqrt(a**2 + b**2 + c**2)

    # 取绝对值，确保都是正值（地面以上）
    heights = np.abs(distances)
```

**关键点**：
- ✅ **直接使用您在"地面识别"中找到的地面平面**
- ✅ 计算的是**垂直距离**，不是简单的Z坐标差
- ✅ 即使地面不是完全水平，也能正确计算

### 步骤2: 自适应分层（基于点云密度）

```python
# 计算平均点间距
distances_nn = cloud.compute_nearest_neighbor_distance()
avg_dist = np.mean(distances_nn)

# 每层厚度 = 平均点间距 × 2.5
optimal_layer_thickness = avg_dist * 2.5

# 计算最优分层数（10-50层之间）
optimal_num_layers = max(10, min(50, int(pile_height / optimal_layer_thickness)))
```

**为什么这样设计**：
- 对于**密集点云**（点间距小）：分层多，精度高
- 对于**稀疏点云**（点间距大）：分层少，避免空层
- 您的测试点云（1603点，0.08m高）：自动选择**10层**

### 步骤3: 逐层计算截面积

```python
for i in range(num_layers + 1):
    h = i * layer_thickness

    # 选择该高度以上的所有点
    mask = heights >= h
    layer_points = points[mask]

    # 计算该层的2D投影面积（使用凸包）
    if len(layer_points) >= 3:
        hull_2d = ConvexHull(layer_points[:, :2])  # 只用XY坐标
        area = hull_2d.volume  # 2D凸包的volume就是面积
```

**关键设计**：
- ✅ **"该高度以上的所有点"**：自动处理重叠点
- ✅ 投影到XY平面：消除Z方向的重叠
- ✅ 使用凸包：包络所有点，形成外轮廓

### 步骤4: 梯形法则累加体积

```python
# 梯形法则：(上底 + 下底) / 2 × 高
if i > 0:
    volume += (prev_area + area) / 2 * layer_thickness

prev_area = area
```

**示意图**：
```
层i+1: 面积A[i+1]  ----
                    |  | 梯形体积 = (A[i] + A[i+1])/2 × Δh
层i:   面积A[i]    ----
```

**实现方式**：
```python
if self.ground_plane is not None:
```

**优势**：
- ✅ **完全依赖地面识别结果**：不重新计算地面
- ✅ **支持倾斜地面**：即使地面不水平，也能正确计算垂直距离
- ✅ **统一基准**：所有高度都相对于同一个地面平面

**对比其他方法**：
- ❌ 凸包法：可能包含地面以下的空间
- ❌ 栅格法：如果地面倾斜，计算会有误差

### 特征2: 处理内部重叠点 ✅

**问题**：点云内部存在重叠点（如您的测试点云有0.5%重叠）

**horizontal_section的解决方案**：

#### 方案1: 投影消除Z方向重叠
```python
# 每层只看XY投影
hull_2d = ConvexHull(layer_points[:, :2])  # 忽略Z坐标
```

**效果**：
- 即使同一XY位置有多个不同Z的点（重叠）
- 投影后只保留XY位置，自动去重
- 计算的是**外轮廓面积**，不受内部重叠影响

#### 方案2: "该高度以上"策略
```python
mask = heights >= h  # 选择该高度以上的所有点
```

**效果**：
- 不是"该高度附近的点"（会受重叠影响）
- 而是"该高度以上的所有点"（自动包含所有重叠点）
- 重叠点会一起投影到XY平面，形成统一的外轮廓

**示意图**：
```
侧视图：
  *  *  ← 堆顶
 * * *
* * * *  ← 有重叠点
* * * *
-------  ← 地面

俯视图（XY投影）：
  ***
 *****   ← 所有重叠点投影到同一位置
*******  ← 只计算外轮廓面积
```

## 📊 与其他方法对比

### 凸包法 (convex_hull)

**原理**：计算包裹所有点的最小凸多面体

**问题**：
- ❌ 对于凹陷的煤堆，会"切掉"凹陷部分
- ❌ 不能很好地处理不规则形状
- ❌ 重叠点可能导致凸包形状异常

**horizontal_section的优势**：
- ✅ 逐层计算，能捕捉形状变化
- ✅ 凹陷部分也能正确计算

### 栅格法 (grid)

**原理**：XY平面网格化，每个网格取最大高度

**问题**：
- ❌ 固定网格大小不适应所有点云
- ❌ 稀疏点云会导致网格插值过度
- ❌ 重叠点在同一网格内只取最大值，可能丢失信息

**horizontal_section的优势**：
- ✅ 自适应分层，适应点云密度
- ✅ 使用凸包而非网格，更准确
- ✅ 重叠点通过投影自然处理

### 体素法 (voxel)

**原理**：3D空间划分为小立方体，统计占据的体素

**问题**：
- ❌ 体素大小难以确定
- ❌ 重叠点在同一体素内只计数一次
- ❌ 对稀疏点云不友好

**horizontal_section的优势**：
- ✅ 不需要3D划分，只需2D投影
- ✅ 计算更稳定
- ✅ 对稀疏点云鲁棒
