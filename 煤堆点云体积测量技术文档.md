## 系统概述

### 1.1 项目背景

本系统是基于DUSt3R三维重建算法获取的点云数据，实现不规则煤堆体积的精确测量。系统采用点云处理技术，通过预处理、地面分割、料堆识别和体积计算等步骤，实现自动化的料堆体积测量。

### 1.2 系统特点

- ✅ **高精度**: 基于点云数据的精确测量，误差可控制在5%以内
- ✅ **自动化**: 全流程自动处理，减少人工干预
- ✅ **可视化**: 提供友好的Web界面和实时可视化
- ✅ **灵活性**: 支持多种体积计算方法和参数调整
- ✅ **可扩展**: 支持多料堆场景的批量处理

### 1.3 技术栈

- **点云处理**: Open3D
- **数值计算**: NumPy, SciPy
- **Web界面**: Gradio
- **3D重建**: DUSt3R
- **可视化**: Matplotlib, Open3D Visualizer

### 3.2 各步骤详细说明

#### 3.2.1 点云加载

**目的**: 读取DUSt3R生成的点云文件，提取基本信息

**输入**: coal_pile.ply文件

**处理**:
- 使用Open3D读取PLY格式文件
- 提取点坐标 (x, y, z)
- 提取颜色信息 (如果有)
- 计算边界框 (min, max, size)
- 统计点数量

**输出**: 点云对象 + 基本信息字典

**代码示例**:
```python
point_cloud = o3d.io.read_point_cloud("coal_pile.ply")
points = np.asarray(point_cloud.points)
info = {
    "点数量": len(points),
    "边界框": {
        "最小值": points.min(axis=0),
        "最大值": points.max(axis=0)
    }
}
```

#### 3.2.2 点云预处理

**目的**: 提高点云质量，减少噪声，优化后续处理

**3.2.2.1 体素下采样 (Voxel Downsampling)**

- **原理**: 将空间划分为规则的体素网格，每个体素内的点用中心点代替
- **参数**: voxel_size (体素大小，单位：米)
- **效果**: 减少点数，保持整体形状
- **建议值**: 0.005-0.01米

```python
downsampled = point_cloud.voxel_down_sample(voxel_size=0.01)
```

**3.2.2.2 统计离群点去除 (Statistical Outlier Removal)**

- **原理**: 计算每个点到其邻居的平均距离，去除距离异常的点
- **参数**: 
  - nb_neighbors: 邻居数量 (建议: 20)
  - std_ratio: 标准差比率 (建议: 2.0)
- **效果**: 去除孤立的噪声点

```python
cl, ind = downsampled.remove_statistical_outlier(
    nb_neighbors=20,
    std_ratio=2.0
)
clean_cloud = downsampled.select_by_index(ind)
```

**3.2.2.3 法向量估计 (Normal Estimation)**

- **原理**: 基于局部邻域拟合平面，计算每个点的法向量
- **用途**: 为后续的表面重建和分割提供几何信息
- **参数**: 搜索半径或邻居数量

```python
clean_cloud.estimate_normals(
    search_param=o3d.geometry.KDTreeSearchParamHybrid(
        radius=0.02,
        max_nn=30
    )
)
```

#### 3.2.3 地面平面分割

**目的**: 将料堆从地面分离，为体积计算提供基准面

**算法**: RANSAC (Random Sample Consensus)

**原理**:
1. 随机选择3个点拟合平面
2. 计算所有点到平面的距离
3. 统计距离小于阈值的点数 (内点数)
4. 重复多次，选择内点数最多的平面
5. 使用所有内点重新拟合平面

**参数**:
- distance_threshold: 点到平面的距离阈值 (建议: 0.01-0.03米)
- ransac_n: 采样点数 (固定为3，平面需要3个点)
- num_iterations: 迭代次数 (建议: 1000-5000)

**输出**:
- 地面平面方程: ax + by + cz + d = 0
- 地面点云 (内点)
- 料堆点云 (外点)

```python
plane_model, inliers = processed_cloud.segment_plane(
    distance_threshold=0.02,
    ransac_n=3,
    num_iterations=1000
)
[a, b, c, d] = plane_model  # 平面方程系数
```

**地面高度计算**:
对于料堆中心点 (x₀, y₀)，其在地面平面上的投影高度为:
```
z_ground = -(a*x₀ + b*y₀ + d) / c
```

#### 3.2.4 料堆聚类 (可选)

**目的**: 在多料堆场景中，识别和分离不同的料堆

**算法**: DBSCAN (Density-Based Spatial Clustering)

**原理**:
1. 对每个点，找到半径eps内的所有邻居
2. 如果邻居数 >= min_points，标记为核心点
3. 核心点及其邻居形成一个簇
4. 重复直到所有点被访问

**参数**:
- eps: 邻域半径 (建议: 0.05-0.2米，取决于料堆间距)
- min_points: 最小点数 (建议: 100-500)

**输出**:
- 簇标签数组 (每个点的簇ID)
- 各簇的点云对象
- 各簇的统计信息 (点数、中心、边界框)

```python
labels = np.array(pile_cloud.cluster_dbscan(
    eps=0.05,
    min_points=100
))
```

**何时使用聚类**:
- ✅ 场景中有多个独立的料堆
- ✅ 料堆之间有明显间隔
- ❌ 只有一个料堆时不需要聚类
- ❌ 料堆紧密相连时聚类效果不佳

#### 3.2.5 体积计算

**目的**: 计算料堆的实际体积

**方法对比**:

| 方法 | 原理 | 优点 | 缺点 | 适用场景 |
|------|------|------|------|----------|
| 凸包法 | 计算点云的凸包体积 | 速度快，实现简单 | 会高估体积 (凸包包含凹陷部分) | 形状规则的料堆 |
| 网格法 | 将XY平面划分为网格，计算每个网格柱的体积 | 精确，适应不规则形状 | 计算量大，受网格大小影响 | 形状不规则的料堆 |
| Alpha Shape | 基于Delaunay三角剖分的精确表面重建 | 最精确 | 实现复杂，计算量大 | 高精度要求场景 |

**3.2.5.1 凸包法 (Convex Hull)**

**步骤**:
1. 将料堆点投影到地面，生成地面投影点
2. 合并料堆点和地面投影点
3. 计算合并后点云的凸包
4. 凸包的体积即为料堆体积

```python
# 添加地面投影点
ground_points = points.copy()
ground_points[:, 2] = ground_z

# 合并点云
all_points = np.vstack([points, ground_points])

# 计算凸包
temp_cloud = o3d.geometry.PointCloud()
temp_cloud.points = o3d.utility.Vector3dVector(all_points)
hull, _ = temp_cloud.compute_convex_hull()
volume = hull.get_volume()
```

**3.2.5.2 网格法 (Grid-based)**

**步骤**:
1. 在XY平面创建规则网格
2. 对每个网格单元，找到其中的所有点
3. 计算该单元内的最大高度
4. 单元体积 = 网格面积 × (最大高度 - 地面高度)
5. 总体积 = 所有单元体积之和

```python
# 创建网格
x_bins = np.arange(x_min, x_max + grid_size, grid_size)
y_bins = np.arange(y_min, y_max + grid_size, grid_size)

volume = 0.0
for i in range(len(x_bins) - 1):
    for j in range(len(y_bins) - 1):
        # 找到网格内的点
        mask = (
            (points[:, 0] >= x_bins[i]) & (points[:, 0] < x_bins[i+1]) &
            (points[:, 1] >= y_bins[j]) & (points[:, 1] < y_bins[j+1])
        )
        if mask.any():
            max_z = points[mask, 2].max()
            height = max(0, max_z - ground_z)
            cell_volume = grid_size * grid_size * height
            volume += cell_volume
```

**参数选择**:
- grid_size: 网格大小 (建议: 0.05-0.1米)
- 较小的网格: 更精确，但计算量大
- 较大的网格: 计算快，但可能损失精度

**3.2.5.3 堆顶位置和高度计算**

```python
# 堆顶位置 (Z坐标最大的点)
max_z_idx = points[:, 2].argmax()
peak_position = points[max_z_idx]

# 地面高度 (基于地面平面方程)
center_xy = points[:, :2].mean(axis=0)
ground_z = -(a * center_xy[0] + b * center_xy[1] + d) / c

# 料堆高度
pile_height = peak_position[2] - ground_z
```

## 算法原理

### 4.1 RANSAC平面拟合算法

**RANSAC (Random Sample Consensus)** 是一种鲁棒的参数估计方法，特别适合处理含有大量离群点的数据。

**算法流程**:

```
输入: 点云P, 距离阈值t, 迭代次数N
输出: 最佳平面模型M

1. 初始化: best_inliers = 0, best_model = None

2. 重复N次:
   a. 随机选择3个点 p1, p2, p3
   b. 拟合平面: 计算法向量 n = (p2-p1) × (p3-p1)
   c. 归一化: n = n / ||n||
   d. 计算平面方程: ax + by + cz + d = 0
      其中 (a,b,c) = n, d = -n·p1
   e. 计算内点数: 
      inliers = 0
      for each point p in P:
          distance = |a*p.x + b*p.y + c*p.z + d| / sqrt(a² + b² + c²)
          if distance < t:
              inliers += 1
   f. 如果 inliers > best_inliers:
      best_inliers = inliers
      best_model = (a, b, c, d)

3. 使用所有内点重新拟合平面 (最小二乘法)

4. 返回 best_model
```

**数学原理**:

平面方程: **ax + by + cz + d = 0**

其中:
- (a, b, c) 是平面的法向量
- d 是平面到原点的距离 (带符号)

点 (x₀, y₀, z₀) 到平面的距离:
```
distance = |ax₀ + by₀ + cz₀ + d| / √(a² + b² + c²)
```

**参数影响**:
- **distance_threshold**: 
  - 过小: 内点太少，可能无法找到平面
  - 过大: 包含非地面点，影响精度
  - 建议: 0.01-0.03米 (取决于点云精度)

- **num_iterations**:
  - 过少: 可能找不到最佳平面
  - 过多: 计算时间长，但提高成功率
  - 建议: 1000-5000次

**成功概率**:
假设离群点比例为ε，采样n个点，迭代k次，成功概率为:
```
P(success) = 1 - (1 - (1-ε)ⁿ)ᵏ
```

例如: ε=0.3 (30%离群点), n=3, k=1000
```
P(success) = 1 - (1 - 0.343)¹⁰⁰⁰ ≈ 100%
```

### 4.2 DBSCAN聚类算法

**DBSCAN (Density-Based Spatial Clustering of Applications with Noise)** 是一种基于密度的聚类算法，能够发现任意形状的簇，并自动识别噪声点。

**核心概念**:

1. **ε-邻域**: 点p的ε-邻域是所有距离p不超过ε的点的集合
   ```
   N_ε(p) = {q ∈ D | dist(p,q) ≤ ε}
   ```

2. **核心点**: 如果点p的ε-邻域内至少有MinPts个点，则p是核心点
   ```
   |N_ε(p)| ≥ MinPts
   ```

3. **边界点**: 不是核心点，但在某个核心点的ε-邻域内

4. **噪声点**: 既不是核心点也不是边界点

**算法流程**:

```
输入: 点集D, 半径ε, 最小点数MinPts
输出: 簇集合C

1. 初始化: 所有点标记为未访问, C = {}

2. for each 未访问的点 p in D:
   a. 标记p为已访问
   b. 找到p的ε-邻域: N = N_ε(p)
   c. if |N| < MinPts:
      标记p为噪声点
   d. else:
      创建新簇 c
      将p加入c
      for each 点 q in N:
         if q未访问:
            标记q为已访问
            N' = N_ε(q)
            if |N'| >= MinPts:
               N = N ∪ N'  # 扩展邻域
         if q不属于任何簇:
            将q加入c
      将c加入C

3. 返回C
```

**参数选择**:

**ε (邻域半径)**:
- 物理意义: 认为属于同一料堆的最大点间距
- 选择方法:
  1. 计算所有点到其第k近邻的距离
  2. 按距离排序并绘制曲线
  3. 选择曲线"肘部"对应的距离
- 建议值: 0.05-0.2米 (取决于料堆间距)

**MinPts (最小点数)**:
- 物理意义: 形成料堆所需的最小点数
- 选择方法:
  - 一般取 MinPts ≥ 维度 + 1 (3D点云: MinPts ≥ 4)
  - 考虑噪声比例: MinPts = ln(n) (n为总点数)
- 建议值: 100-500 (取决于点云密度)

**优点**:
- ✅ 能发现任意形状的簇
- ✅ 自动识别噪声点
- ✅ 不需要预先指定簇数量
- ✅ 对参数相对不敏感

**缺点**:
- ❌ 对密度差异大的数据效果不佳
- ❌ 高维数据性能下降
- ❌ 参数选择需要经验

### 4.3 体积计算数学原理

#### 4.3.1 凸包体积计算

**凸包定义**: 包含所有点的最小凸集

**体积计算**:
1. 构建Delaunay三角剖分
2. 计算每个四面体的体积
3. 求和得到总体积

**四面体体积公式**:
给定四个顶点 p₁, p₂, p₃, p₄:
```
V = |det([p₂-p₁, p₃-p₁, p₄-p₁])| / 6
```

其中det表示行列式:
```
det = (p₂-p₁)·((p₃-p₁) × (p₄-p₁))
```

**误差分析**:
- 凸包体积 ≥ 实际体积
- 误差来源: 凸包填充了料堆的凹陷部分
- 对于规则形状 (锥形、半球形): 误差 < 5%
- 对于不规则形状: 误差可能 > 10%

#### 4.3.2 网格体积计算

**原理**: 将料堆分解为垂直柱体的集合

**体积公式**:
```
V = Σᵢⱼ Aᵢⱼ × hᵢⱼ
```

其中:
- Aᵢⱼ: 网格单元(i,j)的面积 = grid_size²
- hᵢⱼ: 网格单元(i,j)的高度 = max(zₖ) - z_ground
  (zₖ为该单元内所有点的z坐标)

**精度分析**:

网格大小对精度的影响:
- grid_size → 0: 精度 → 100% (但计算量 → ∞)
- grid_size过大: 损失细节，低估体积

**误差估计**:
假设料堆边界的平均斜率为α，网格大小为g:
```
相对误差 ≈ g × tan(α) / H
```
其中H为料堆平均高度

例如: g=0.1m, α=45°, H=3m
```
相对误差 ≈ 0.1 × 1 / 3 ≈ 3.3%
```

**优化策略**:
1. **自适应网格**: 边界区域使用更小的网格
2. **插值**: 使用邻近点插值估计网格高度
3. **边界修正**: 对边界网格进行特殊处理

#### 4.3.3 体积不确定度分析

**误差来源**:

1. **点云采集误差** (σ_scan):
   - DUSt3R重建精度: ±0.01-0.05m
   - 影响: 直接影响高度测量

2. **地面拟合误差** (σ_ground):
   - RANSAC拟合残差: ±0.01-0.03m
   - 影响: 影响基准面确定

3. **体积计算误差** (σ_method):
   - 凸包法: +5% ~ +15%
   - 网格法: ±3% ~ ±5%

**总不确定度**:
```
σ_total = √(σ_scan² + σ_ground² + σ_method²)
```

**示例计算**:
假设:
- σ_scan = 0.02m
- σ_ground = 0.02m
- 料堆底面积 A = 100m²
- 平均高度 H = 3m
- 使用网格法: σ_method = 4%

高度不确定度:
```
σ_H = √(σ_scan² + σ_ground²) = √(0.02² + 0.02²) = 0.028m
```

体积不确定度:
```
V = A × H = 100 × 3 = 300 m³
σ_V = V × √((σ_H/H)² + σ_method²)
    = 300 × √((0.028/3)² + 0.04²)
    = 300 × √(0.0001 + 0.0016)
    = 300 × 0.041
    = 12.3 m³
```

相对不确定度: 12.3/300 = 4.1%

**结论**: 在良好条件下，体积测量的相对不确定度可控制在5%以内。
