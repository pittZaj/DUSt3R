## 一、项目概述

### 1.1 核心需求
**场景**：洗煤厂煤堆体积自动测量，替代人工测量
**输入**：多角度拍摄的煤堆照片（5-10张）
**输出**：煤堆体积（m³）、重量（吨）、3D点云模型

### 1.2 关键约束
- ❌ **无相机标定**：现场拍摄，无法获取相机内参
- ❌ **无已知尺度**：缺少参考物体或距离信息
- ✅ **多视角图像**：可围绕煤堆拍摄多角度照片
- ✅ **地面参考**：每张照片包含地面

## 二、当前技术方案

### 2.1 技术路线
```
多视角图像 → 3D重建 → 点云提取 → 地面分割 → 体积计算 → 尺度标定
```

### 2.2 核心算法
| 模块 | 算法 | 说明 |
|------|------|------|
| **3D重建** | DUSt3R | 无需相机标定的单目3D重建 |
| **点云提取** | 置信度过滤 | 阈值：0.5（可调） |
| **点云处理** | 统计离群点去除 + 体素下采样 | std_ratio=3.0, voxel=0.01m |
| **地面分割** | 高度阈值法（推荐）/ RANSAC平面拟合 | 简单快速 / 适用倾斜地面 |
| **体积计算** | Delaunay + 凸包 + 体素化（融合） | 多方法投票，提高鲁棒性 |

### 2.3 系统架构
```python
# 处理流程
1. load_images() → 加载并resize图像（512×512）
2. make_pairs() → 生成图像对（完全图）
3. inference() → DUSt3R推理，获取深度和置信度
4. global_aligner() → 全局对齐（MST初始化，300次迭代）
5. extract_pointcloud() → 提取点云（置信度过滤）
6. postprocess_pointcloud() → 去噪 + 下采样
7. segment_coal_pile() → 地面分割
8. calculate_volume() → 体积计算（三种方法融合）
```

## 三、核心技术问题与解决方案

### 3.1 ⚠️ **尺度歧义问题（最关键）**

#### 问题描述
- DUSt3R输出的是**相对尺度**的点云，无法直接获得真实世界尺度
- 例如：点云体积1.5m³，但不知道1个点云单位 = 多少米

#### 当前解决方案
**方法1：参考物体标定（推荐）**
```
1. 拍摄时放置已知尺寸的参考物体（如1米标尺）
2. 在点云中测量参考物体长度
3. 计算尺度因子 = 真实长度 / 点云长度
4. 真实体积 = 点云体积 × (尺度因子)³
```

**方法2：已知距离标定**
```
1. 记录拍摄距离 D
2. 使用相机焦距估算尺度
3. 尺度因子 ≈ D / (焦距 × 图像尺寸)
```

#### 🔴 **问题咨询**
1. **是否有更自动化的尺度恢复方法？**
   - 例如：利用地面平面的先验知识？
   - 或者通过多次测量建立尺度数据库？

2. **如何在无参考物体的情况下获得绝对尺度？**
   - 是否可以通过图像EXIF信息（焦距、传感器尺寸）估算？
   - 或者通过已知的地面网格尺寸？

---

### 3.2 ⚠️ **地面分割精度问题**

#### 问题描述
- 点云中的地面点不在同一平面上（噪声、误差）
- 简单的高度阈值方法可能误分割

#### 当前解决方案
提供两种方法：
1. **简单高度阈值法**（推荐，默认）
   - 使用Z坐标的5%分位数作为地面高度
   - 过滤高度 > ground_height + 2cm 的点
   - 优点：快速、保留点多
   - 缺点：地面倾斜时不准确

2. **RANSAC平面拟合法**（可选）
   - 拟合地面平面方程：ax + by + cz + d = 0
   - 使用平面距离过滤（阈值3cm）
   - 优点：适应倾斜地面
   - 缺点：点数减少、速度慢

#### 🔴 **问题咨询**
1. **如何更准确地分割地面？**
   - 是否应该使用语义分割网络预先识别地面区域？
   - 或者结合颜色、纹理信息？

2. **RANSAC参数如何自适应调整？**
   - 当前距离阈值固定为3cm，是否合理？
   - 如何根据点云密度自动调整？

---

### 3.3 ⚠️ **点云质量不稳定**

#### 问题描述
- 从网站截图的3D模型渲染图无法重建（置信度极低）
- 真实照片的重建效果依赖图像质量和重叠度

#### 当前解决方案
- 提供可调参数：置信度阈值、离群点过滤强度、下采样体素大小
- 推荐参数：confidence=0.5, outlier_std=3.0, voxel=0.01m

#### 🔴 **问题咨询**
1. **如何评估输入图像的质量？**
   - 能否在处理前预判图像是否适合重建？
   - 例如：检测重叠度、光照一致性、运动模糊等

2. **如何提高点云密度和质量？**
   - 是否应该使用更大的图像尺寸（当前512px）？
   - 或者调整DUSt3R的推理参数？

---

### 3.4 ⚠️ **体积计算准确性**

#### 问题描述
- 三种方法（Delaunay、凸包、体素化）结果差异较大
- 当前使用简单平均融合

#### 当前解决方案
```python
# 多方法融合
v1 = delaunay_volume()      # 三角剖分
v2 = convex_hull_volume()   # 凸包
v3 = voxel_volume()         # 体素化

volume = mean([v1, v2, v3])  # 简单平均
confidence = 1 / (1 + std/mean)  # 基于一致性的置信度
```

#### 🔴 **问题咨询**
1. **哪种体积计算方法最适合煤堆场景？**
   - 煤堆形状不规则，凸包法会高估
   - Delaunay对噪声敏感
   - 体素化精度依赖体素大小

2. **如何更合理地融合多种方法？**
   - 是否应该使用加权平均？
   - 如何根据点云质量动态选择方法？

## 六、技术选型疑问

### 6.1 DUSt3R vs 其他方案

| 方案 | 优点 | 缺点 | 是否考虑 |
|------|------|------|---------|
| **DUSt3R**（当前） | 无需标定、快速 | 相对尺度、对渲染图失效 | ✅ 使用中 |
| **COLMAP** | 精度高、开源 | 需要特征匹配、速度慢 | ❓ 备选 |
| **NeRF/3DGS** | 视觉效果好 | 训练慢、需要密集视角 | ❌ 不适合 |
| **单目深度估计** | 单张图像即可 | 尺度歧义更严重 | ❌ 不适合 |
| **双目/RGB-D** | 绝对尺度 | 需要特殊硬件 | ❓ 未来考虑 |

#### 🔴 **问题咨询**
1. **DUSt3R是否是最佳选择？**
   - 对于煤堆测量场景，是否有更合适的算法？
   - COLMAP的精度提升是否值得牺牲速度？

2. **是否应该考虑硬件升级？**
   - 使用双目相机或RGB-D相机（如RealSense）？
   - 或者使用激光雷达（LiDAR）？
