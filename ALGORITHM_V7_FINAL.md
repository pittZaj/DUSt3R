# 物料堆地面识别算法 - 最终优化方案

## 一、需求背景

### 1.1 项目目标（来自README.md）
- **应用场景**：洗煤厂/钢厂的库存监管，煤堆体积测算
- **核心需求**：从多角度拍摄的照片重建3D点云，准确识别地面并计算体积
- **关键约束**：无相机标定、无已知尺度、DUSt3R重建点云（表面点为主）

### 1.2 物料堆特性
- **几何特征**：煤堆堆放在地面上，地面是最低的、水平的支撑面
- **点云特点**：DUSt3R重建的点云主要是表面点，地面点极少（<10%）
- **识别难点**：侧面底部可能也很平坦，容易误判为地面

## 二、博主推荐方法分析

参考文档：`/mnt/data3/clip/DUSt3R/test/点云处理-地面点提取方法.md`

### 2.1 三种方法对比

| 方法 | 原理 | 适用场景 | 是否适合我们 |
|------|------|----------|--------------|
| 直接计算法（lego_loam） | 计算相邻激光线俯仰角 | 激光雷达数据 | ❌ 不适合（我们是RGB重建） |
| 平面拟合法 | SVD分解协方差矩阵 | 地面点较多的场景 | ⚠️ 部分适用 |
| 滤除法（hdl_graph_slam） | 法线过滤 + RANSAC + 验证 | 激光雷达 + 已知传感器高度 | ✅ 最适合（需改进） |

### 2.2 选择滤除法的原因

**hdl_graph_slam的滤除法**核心思路：
1. 根据传感器高度进行高度范围取点
2. 根据法线与Z轴夹角过滤（地面法线应该向上）
3. 使用RANSAC拟合地平面
4. 验证法线是否与Z轴夹角<10°（水平度验证）

**优势**：
- ✅ 利用法线信息（地面法线向上）
- ✅ 水平度强验证（<10°）
- ✅ RANSAC鲁棒拟合

**需要改进的地方**：
- ❌ 依赖传感器高度（我们没有）
- ❌ 针对激光雷达（我们是RGB重建）
- ❌ 没有考虑覆盖率和中心性

## 三、物料堆专用优化方案

### 3.1 算法设计

结合博主推荐的滤除法和物料堆特性，设计**法线验证 + 多策略融合**算法：

```
算法流程：

第一步：法线预过滤
- 计算所有点的法向量
- 只保留法向量向上的点（与Z轴夹角<60°）
- 这些点更可能是地面点（地面法线向上）

第二步：Z值分层尝试
策略A：法线过滤 + Z值分层
  - 在法线向上的点中，按Z值排序
  - 尝试底部5%, 8%, 10%, 12%, 15%, 20%, 25%
  - 对每个百分比，使用RANSAC拟合平面
  - 验证平面法线与Z轴夹角<10°（水平度强约束）

策略B：直接Z值分层（备选）
  - 不使用法线过滤，直接按Z值排序
  - 尝试底部5%, 8%, 10%
  - 同样验证水平度<10°

第三步：综合评分
对每个候选平面计算：
- 水平度得分 = 1 - angle/10°（50%权重）
- 覆盖率得分 = coverage/30%（30%权重）
- 中心性得分 = 1 - offset/25%（20%权重）

第四步：选择最佳
- 选择综合得分最高的平面
- 如果没有找到水平平面，回退到全局RANSAC
```

### 3.2 关键改进点

#### 改进1：法线预过滤（来自博主推荐）
```python
# 地面点的法向量应该向上（与Z轴夹角<60°）
z_axis = np.array([0, 0, 1])
normal_angles = np.degrees(np.arccos(np.abs(normals @ z_axis)))
upward_mask = normal_angles < 60.0
```

**物理意义**：地面是水平支撑面，法向量应该向上。侧面的法向量是水平的，会被过滤掉。

#### 改进2：水平度强约束（来自博主推荐）
```python
# 只接受接近水平的平面（<10°）
if angle > 10.0:
    continue
```

**物理意义**：地面必须接近水平，这是硬约束。侧面即使很平坦，倾角也通常>10°。

#### 改进3：综合评分（针对物料堆优化）
```python
# 权重：水平度50% + 覆盖率30% + 中心性20%
score = horizontal_score * 0.5 + coverage_score * 0.3 + center_score * 0.2
```

**设计理由**：
- 水平度最重要（50%）：地面必须水平
- 覆盖率次之（30%）：地面应该覆盖整个区域，但DUSt3R地面点少，降低阈值到30%
- 中心性最后（20%）：地面应该在中心，但不是最关键的

#### 改进4：多策略融合（增加鲁棒性）
- 策略A：法线过滤 + Z值分层（优先）
- 策略B：直接Z值分层（备选）

**设计理由**：如果法线估计不准确，仍然可以通过直接Z值找到地面。

### 3.3 参数设置

| 参数 | 值 | 物理意义 |
|------|-----|----------|
| 法线角度阈值 | 60° | 地面法线与Z轴夹角<60°认为向上 |
| 水平度阈值 | 10° | 地面平面与水平面夹角<10° |
| 覆盖率阈值 | 30% | 地面应覆盖XY投影面积的30%以上 |
| 中心偏移阈值 | 25% | 地面中心与点云中心偏移<25% |
| Z值百分比 | 5-25% | 尝试底部5%, 8%, 10%, 12%, 15%, 20%, 25% |

### 4.2 结果分析

**为什么选择"直接Z值+底部5%"而不是"法线过滤+底部5%"？**

| 指标 | 法线过滤+5% | 直接Z值+5% | 分析 |
|------|-------------|------------|------|
| 倾角 | 0.2° | 0.1° | 直接Z值更水平 |
| 覆盖率 | 27.7% | 32.7% | 直接Z值覆盖更好 |
| 点数 | 42 | 59 | 直接Z值点更多 |
| 得分 | 0.824 | 0.870 | 直接Z值综合更优 |

**结论**：法线过滤虽然有物理意义，但可能过滤掉了一些真正的地面点。直接Z值方法在这个案例中效果更好。

**多策略融合的价值**：如果只用法线过滤，可能会错过最佳平面。多策略确保了鲁棒性。

## 六、工程化建议

### 6.1 使用方法

#### Web界面
1. 访问 http://localhost:7869
2. 上传点云文件（推荐5-10张多角度照片重建）
3. 执行预处理：
   - 体素大小：0.002m
   - 标准差倍数：4.0
   - 开启分层剔除：是
4. 选择地面识别方法：**coordinate_correction**
5. 查看日志确认：倾角<5°，覆盖率>25%

#### Python代码
```python
from coal_pile_volume_processor import CoalPileVolumeProcessor

processor = CoalPileVolumeProcessor()
processor.load_point_cloud("coal_pile.ply")
processor.preprocess_point_cloud(
    voxel_size=0.002,
    nb_neighbors=15,
    std_ratio=4.0,
    remove_layers=True
)

result = processor.segment_ground_plane(
    distance_threshold=0.01,
    method="coordinate_correction",
    keep_all_points=True
)

print(f"地面倾角: {result['地面倾角']}°")
```

### 6.2 预期效果

- **地面倾角**：<3°（通常<1°）
- **覆盖率**：25-40%（DUSt3R地面点少，这是正常的）
- **鲁棒性**：法线验证 + 多策略融合，适应不同场景

### 6.3 故障排查

| 现象 | 可能原因 | 解决方案 |
|------|----------|----------|
| 倾角>10° | 点云质量差/地面点太少 | 增加拍摄角度，确保地面可见 |
| 覆盖率<20% | 地面点极少 | 正常现象，只要倾角<5°即可 |
| 未找到水平平面 | 点云严重倾斜 | 检查拍摄角度，确保包含地面 |
