### 2. 核心功能实现

#### 2.1 CoalVolumeEstimator类

**主要方法**:

1. **`reconstruct_3d()`** - 3D重建
   - 加载多张图像
   - 生成图像对
   - 执行DUSt3R推理
   - 全局对齐优化

2. **`extract_pointcloud()`** - 点云提取
   - 从场景中提取3D点
   - 应用置信度阈值过滤
   - 合并多视图点云

3. **`postprocess_pointcloud()`** - 点云后处理
   - 离群点去除
   - 体素下采样
   - 法向量估计

4. **`segment_coal_pile()`** - 煤堆分割
   - 估计地面高度
   - 过滤地面点
   - 提取煤堆区域

5. **`calculate_volume()`** - 体积计算
   - **Delaunay三角剖分法**
   - **凸包法**
   - **体素化方法**
   - 多方法融合与置信度评估

6. **`estimate_weight()`** - 重量估算
   - 根据体积和密度计算重量
   - 支持自定义煤炭密度

### 4. 关键技术实现

#### 4.1 3D重建流程

```
图像输入 → 图像对生成 → DUSt3R推理 → 全局对齐 → 点云提取
```

**技术特点**:
- ✨ 无需相机标定
- ⚡ GPU加速推理
- 🎯 自动全局对齐
- 📈 置信度评估

#### 4.2 体积计算方法

**方法1: Delaunay三角剖分**
- 添加地面网格点
- 构建Delaunay四面体网格
- 累加四面体体积

**方法2: 凸包法**
- 计算点云凸包
- 直接获取凸包体积
- 适用于规则形状

**方法3: 体素化**
- 将点云体素化
- 统计地面以上体素数
- 计算总体积

**融合策略**:
- 取多种方法的平均值
- 基于标准差计算置信度
- 过滤无效结果

#### 4.3 性能优化

- 🚀 TF32加速（GPU支持时）
- 💾 GPU内存优化
- 🔄 自动内存清理
- ⏱️ 详细计时统计

## 📊 使用流程

### 1. 准备图像

**要求**:
- 至少2张图像（推荐5-10张）
- 多角度拍摄（正面、侧面、顶部）
- 图像间50-70%重叠
- 分辨率≥1920x1080
- 避免运动模糊

### 2. 上传图像

- 点击"上传煤堆图像"区域
- 选择多张图像文件
- 支持JPG、PNG等格式

### 3. 设置参数

**煤炭密度**:
- 原煤: 1200-1400 kg/m³
- 精煤: 1300-1500 kg/m³
- 默认: 1300 kg/m³

**图像尺寸**:
- 224px: 快速但精度较低
- 512px: 推荐，平衡速度和精度
- 更大尺寸需要更多显存

### 4. 开始处理

- 点击"🚀 开始处理"按钮
- 等待处理完成（可能需要几分钟）
- 查看结果

### 5. 下载结果

- **点云文件** (.ply): 可用Open3D或CloudCompare查看
- **结果文件** (.json): 包含详细测算数据

## ⚠️ 注意事项

### 1. 首次使用

- 首次处理可能需要较长时间（模型加载）
- 建议使用测试图像验证系统

### 2. 图像质量

- 确保图像清晰、光线充足
- 避免逆光、运动模糊
- 多角度覆盖煤堆

### 3. 结果准确性

- 结果为预估值，非精确测量
- 建议使用已知体积的煤堆进行校准
- 置信度越高，结果越可靠

### 4. 性能考虑

- 图像数量越多，处理时间越长
- 图像尺寸越大，显存占用越多
- 建议根据实际需求平衡精度和速度
